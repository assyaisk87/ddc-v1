
Ты senior Angular frontend developer и UI/UX engineer. Нужно доработать существующий Angular-сайт ЦЦР в футуристическом стиле dark/neon/glassmorphism.

Цель: реализовать новый первый экран сайта в формате AI Command Center + интерактивная карта цифровой экосистемы ЦЦР.

ВАЖНО:
Не ломай существующий проект.
Сначала изучи структуру проекта.
Используй уже существующие цвета, переменные, стили, компоненты и подход к дизайну.
Если в проекте есть theme.scss, styles.scss, shared styles — используй их.
Не удаляй существующий AI Assistant, а красиво интегрируй его в новый hero-блок.

ЗАДАЧА 1. Проанализируй проект

1. Найди главный компонент главной страницы:

   * home.component.html
   * home.component.scss
   * home.component.ts
     или аналогичный компонент.

2. Найди текущие глобальные стили:

   * styles.scss
   * _theme.scss
   * variables.scss
   * mixins.scss

3. Определи текущие цвета проекта:

   * dark background
   * cyan / blue neon
   * purple neon
   * accent pink
   * glass cards
   * gradients

4. Не меняй архитектуру без необходимости.

ЗАДАЧА 5. Стили

Используй стиль:

* dark background
* glassmorphism
* neon cyan
* neon purple
* subtle gradient
* rounded corners 20–28px
* soft shadows
* border: 1px solid rgba(...)
* backdrop-filter: blur(...)
* плавные transition 0.3s ease

Не делай слишком много свечения. Дизайн должен выглядеть дорого и современно, а не как перегруженный crypto-сайт.

Пример цветов, если в проекте нет переменных:

* background: #070A12
* surface: rgba(15, 23, 42, 0.72)
* cyan: #00D4FF
* purple: #7B2CBF
* blue: #3B82F6
* green: #00FFA3
* text: #F8FAFC
* muted text: #94A3B8

ЗАДАЧА 6. Анимации

Добавь:

* плавное появление hero-контента при загрузке;
* floating animation для узлов;
* pulse animation для центрального узла ЦЦР;
* animated SVG lines;
* hover glow;
* prefers-reduced-motion поддержку.

Если пользователь включил reduced motion, анимации должны быть отключены или сильно упрощены.

ЗАДАЧА 7. Адаптивность

Desktop:

* hero в 2 колонки;
* слева текст, справа ecosystem map / command center.

Tablet:

* hero в 1 колонку;
* карта ниже текста;
* метрики в 2 колонки.

Mobile:

* заголовок не должен ломаться;
* узлы экосистемы можно расположить вертикальными карточками;
* SVG-линии можно скрыть;
* кнопки во всю ширину;
* метрики в 1 колонку или 2 маленькие колонки.

ЗАДАЧА 8. Angular-реализация

В TypeScript создай массивы данных:

ecosystemNodes = [
{
title: 'AI Сервисы',
icon: '...',
description: 'Интеллектуальные сервисы для автоматизации и поддержки цифровых процессов.',
status: 'В развитии'
},
...
]

commandMetrics = [
{
value: '27+',
label: 'Активных проектов',
icon: '...'
},
...
]

Не хардкодь одинаковые карточки вручную.
Используй *ngFor или @for, в зависимости от версии Angular в проекте.

ЗАДАЧА 9. Необходимо проверить

После реализации:

1. Запусти проект локально.
2. Проверь, что нет ошибок Angular/Sass.
3. Проверь desktop, tablet, mobile.
4. Проверь, что существующие секции сайта не сломались.
5. Проверь, что AI Assistant остался рабочим.
6. Проверь Lighthouse хотя бы визуально:

   * нет сильного layout shift;
   * изображения не ломают загрузку;
   * анимации не тормозят.
